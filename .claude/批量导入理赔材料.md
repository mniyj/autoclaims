# 离线材料导入功能实施计划

## Context

理赔员需要在赔案详情页面导入线下收集的理赔材料。系统应自动对材料进行分类、内容识别，并判断材料是否齐全，不齐全时提示缺失项。

 **目标页面** : [ClaimCaseDetailPage.tsx](components/ClaimCaseDetailPage.tsx)

 **现有功能复用** :

* 47种材料类型定义: [claims-materials.json](jsonlist/claims-materials.json)
* 文件处理服务: [fileProcessor.js](server/services/fileProcessor.js)
* 多文件分析: [multiFileAnalyzer.js](server/services/multiFileAnalyzer.js)
* 基础Modal组件: [Modal.tsx](components/ui/Modal.tsx)
* 浮动按钮模式: [AITestButton.tsx](components/AITestButton.tsx)

---

## Implementation Steps

### Step 1: 新增类型定义

 **文件** : [types.ts](types.ts)

```typescript
// 离线材料导入相关类型
export interface OfflineMaterialImportResult {
  success: boolean;
  importedFiles: ProcessedFile[];
  completeness: CompletenessResult;
  warnings: string[];
}
export interface ProcessedFile {
  documentId: string;
  fileName: string;
  fileType: string;
  ossUrl?: string;
  extractedText?: string;
  structuredData?: Record<string, unknown>;
  classification: {
    materialId: string;
    materialName: string;
    confidence: number;
  };
  status: 'processing' | 'completed' | 'failed';
}
export interface CompletenessResult {
  isComplete: boolean;
  score: number;
  requiredMaterials: string[];
  providedMaterials: string[];
  missingMaterials: string[];
  warnings: string[];
}
```

### Step 3: 创建导入对话框组件

 **文件** : `components/OfflineMaterialImportDialog.tsx`

 **UI结构** :

```warp-runnable-command
┌─────────────────────────────────────────────┐
│  离线材料导入                          [×]  │
├─────────────────────────────────────────────┤
│  [拖拽上传区域]                             │
│  或点击选择文件 (支持图片/PDF/Word/Excel)   │
├─────────────────────────────────────────────┤
│  已上传文件:                                │
│  ┌─────────┐ ┌─────────┐ ┌─────────┐       │
│  │ 文件1   │ │ 文件2   │ │ 文件3   │       │
│  │ 医疗发票 │ │ 诊断证明 │ │ 处理中  │       │
│  │ ✓ 已分类 │ │ ✓ 已分类 │ │ ⏳ OCR  │       │
│  └─────────┘ └─────────┘ └─────────┘       │
├─────────────────────────────────────────────┤
│  ⚠️ 材料完整性检查:                         │
│  ✓ 身份证正面                               │
│  ✓ 医疗发票                                 │
│  ✗ 缺少: 银行卡                             │
│  ✗ 缺少: 诊断证明                           │
├─────────────────────────────────────────────┤
│           [取消]  [确认导入]                │
└─────────────────────────────────────────────┘
```

 **功能流程** :

1. 用户拖拽/选择文件
2. 文件上传到OSS
3. 调用后端API进行OCR和分类
4. 显示分类结果，允许手动调整
5. 检查材料完整性
6. 显示缺失材料提示
7. 确认导入后更新赔案文件列表

### Step 4: 新增后端API

 **文件** : [server/apiHandler.js](server/apiHandler.js)

新增 `/api/import-offline-materials` 端点:

```js
// POST /api/import-offline-materials
// Request: { claimCaseId, productCode, files: [{fileName, mimeType, base64Data}] }
// Response: { documents, completeness, summary }
```

 **处理逻辑** :

1. 上传文件到OSS
2. 调用 fileProcessor.js 处理每个文件
3. 使用AI对材料进行分类(基于claims-materials.json)
4. 调用 multiFileAnalyzer.js 检查完整性
5. 返回分类结果和完整性报告

### Step 5: 集成到赔案详情页

 **文件** : [components/ClaimCaseDetailPage.tsx](components/ClaimCaseDetailPage.tsx)

修改内容:

1. 添加状态: `const [showImportDialog, setShowImportDialog] = useState(false)`
2. 添加浮动按钮组件
3. 添加对话框组件
4. 处理导入完成回调，更新文件列表
