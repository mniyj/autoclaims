# FAQ 配置功能产品需求文档 (PRD)

## 1. 概述
FAQ 配置功能旨在为不同的保险产品分类（如“综合意外”、“百万医疗”等）提供一套标准化的常见问题解答配置工具。通过该功能，管理员可以预设针对特定险种的常见疑问及其标准回答，并在前端以结构化的方式展示给最终用户，帮助用户快速理解产品特性、理赔流程等关键信息。

## 2. 用户角色
*   **管理员/运营人员**: 负责在后台配置和维护各险种的 FAQ 列表。
*   **最终用户**: 在产品详情页或投保页查看 FAQ，获取帮助。

## 3. 功能需求

### 3.1 数据结构定义
FAQ 数据作为 `CategoryDefinition` (险种定义) 的一部分存储，字段定义如下：

| 字段名 | 类型 | 必填 | 默认值 | 说明 | 约束 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| `question` | String | 是 | 空字符串 | **问题简称**。展示给用户的核心疑问点。 | 建议不超过 10 个中文字符，保持精简。 |
| `isFocus` | Boolean | 是 | `false` | **是否重点关注**。用于标识该问题是否为用户高频关注点，前端可据此高亮或优先展示。 | - |
| `answer` | String | 是 | 空字符串 | **问题详细**。问题的具体解答内容。 | 支持模板变量语法 `{变量名}`，用于动态替换具体产品的属性值。 |

### 3.2 界面交互与布局
#### 3.2.1 整体布局
*   **位置**: “保险产品配置页面” -> 左侧选择二级或三级险种 -> 右侧“常见问题配置”区域。
*   **展示形式**: 列表式展示，每一条 FAQ 占据一行（Single Row Layout）。
    *   **目的**: 节省垂直屏幕空间，便于管理员在单屏内查看更多配置项。

#### 3.2.2 单条 FAQ 交互组件
每一行 FAQ 配置项包含左右两部分，采用 Flex 布局：

*   **左侧区域 (约占 1/3 宽度, 最小宽度 200px)**:
    *   **顶部**: 
        *   Label: “问题简称”
        *   Control: “重点关注” 复选框 (Checkbox) + 文字说明。
    *   **底部**: 输入框 (Input Type="text")，用于编辑 `question`。
    *   **样式**: 紧凑排列。

*   **右侧区域 (占据剩余宽度)**:
    *   **顶部**: Label: “问题详细”
    *   **底部**: 文本域 (Textarea)。
        *   **行数**: 默认为 1 行 (`rows={1}`)。
        *   **高度**: 固定最小高度 (如 `38px`)。
        *   **缩放**: 禁止用户手动调整大小 (`resize-none`)，以保持界面整洁。

*   **删除操作**:
    *   鼠标悬停在某一行时，右上角显示“删除”图标（垃圾桶）。
    *   点击删除图标，移除该条 FAQ。

#### 3.2.3 列表操作
*   **新增**: 列表底部提供“添加常见问题”按钮，点击后在列表末尾追加一条空记录。
*   **空状态**: 当列表为空时，显示“暂无常见问题配置”的提示占位符。

### 3.3 逻辑约束
1.  **模板变量支持**: `answer` 字段需支持 `{}` 包裹的变量名（如 `{保障范围}`），前端在渲染时应具备识别或替换这些变量的能力（注：当前配置页仅负责存储字符串，替换逻辑在 C 端渲染层实现）。
2.  **数据联动**: 修改任意输入框内容，应实时更新 React 状态 (`formData.faqList`)，确保保存时数据即时生效。

## 4. 技术实现参考

### 4.1 TypeScript 接口
```typescript
export interface FAQItem {
  question: string;  // 问题简称
  answer: string;    // 问题详细解答
  isFocus: boolean;  // 是否重点关注
}

export interface CategoryDefinition {
  // ... 其他现有字段
  faqList?: FAQItem[]; // FAQ 列表
}
```

### 4.2 现有代码结构 (参考)
*   **文件**: `components/InsuranceTypeManagementPage.tsx`
*   **渲染逻辑**: 
    *   使用 `flex` 布局容器。
    *   左侧 `div` class: `w-1/3 min-w-[200px]`
    *   右侧 `div` class: `flex-1`
    *   输入控件绑定 `onChange` 事件更新 `faqList` 数组。

## 5. 验收标准
1.  **布局检查**: FAQ 配置项是否严格保持在一行显示，没有出现不必要的换行。
2.  **功能检查**: 
    *   能否正常输入“问题简称”和“问题详细”。
    *   勾选/取消“重点关注”是否生效。
    *   点击“添加”能新增一行。
    *   点击“删除”能移除指定行。
3.  **数据检查**: 切换险种分类后，FAQ 数据能否正确回显（如从“综合意外”切换到“出行意外”，显示各自独立的 FAQ）。
4.  **持久化**: 修改配置后，数据应能正确反映在最终的配置 JSON/对象中（当前为内存态或文件态，视具体持久化方案而定）。
