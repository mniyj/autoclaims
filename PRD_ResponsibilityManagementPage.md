# 责任管理页面 (ResponsibilityManagementPage) 产品需求文档 (PRD)

## 1. 概述 (Overview)

**责任管理页面** 是保险产品配置系统中的一个核心功能模块，旨在为产品配置人员提供一个统一的“责任字典”维护界面。通过该页面，用户可以管理标准化的保险责任项（如“一般医疗保险金”、“身故保险金”等），支持增删改查操作，并为上层产品配置提供基础数据源。

### 1.1 目标
- 建立标准化的责任库，避免责任定义重复或不一致。
- 提供直观的界面支持责任项的快速检索与维护。
- 确保责任项与险种分类（一级分类）的正确关联。

### 1.2 范围
- **包含**：责任列表展示、模糊搜索、新增责任、编辑责任、删除责任、查看详情。
- **不包含**：责任的具体费率计算逻辑、理赔规则引擎配置、产品级别的责任组合（Plan）配置。

---

## 2. 用户角色 (User Roles)

| 角色 | 权限 | 典型场景 |
| :--- | :--- | :--- |
| **产品运营/配置人员** | 读写 | 维护通用的责任字典，为新产品上线做准备。 |
| **系统管理员** | 读写 | 管理基础配置数据。 |

## 3. 功能需求 (Functional Requirements)

### 3.1 责任列表展示
- **功能描述**：以表格形式展示当前系统中的所有责任项。
- **展示字段**：
  - **序号**：动态生成的行号。
  - **责任代码 (Code)**：唯一标识，如 `GENERAL_MEDICAL`（通常显示为英文/拼音大写）。
  - **责任名称 (Name)**：中文显示名，如 `一般医疗保险金`。
  - **责任分类 (Category)**：所属的一级险种分类，如 `医疗险`、`重疾险`。
  - **责任描述 (Description)**：简要说明，表格中超长需截断展示。
  - **操作**：包含“详情”、“编辑”、“删除”按钮。
- **默认排序**：按创建时间倒序或列表默认顺序。

### 3.2 模糊搜索
- **功能描述**：支持根据关键词快速过滤列表。
- **搜索规则**：
  - 输入框支持输入文本。
  - 匹配范围：**责任代码** 或 **责任名称**。
  - 匹配方式：不区分大小写的包含匹配（Fuzzy Match）。
- **交互**：输入关键词后点击“搜索”按钮或实时过滤（当前实现为点击搜索或依赖 React 状态更新）。

### 3.3 新增责任
- **功能描述**：向字典中添加新的责任项。
- **输入字段**：
  - **责任代码** (必填)：文本输入，建议使用大写英文下划线格式。
  - **责任名称** (必填)：文本输入。
  - **责任分类** (必填)：下拉选择，选项来源于系统一级分类常量（如 `医疗险`, `意外险`, `定期寿险` 等）。
  - **责任描述** (选填)：多行文本输入。
- **校验**：
  - 必填项为空时，“确定”按钮禁用或点击后提示。
  - 保存后自动生成唯一 ID（前端模拟使用时间戳）。

### 3.4 编辑责任
- **功能描述**：修改现有责任项的信息。
- **限制**：
  - **责任代码** 不可修改（作为业务主键，避免破坏引用关系）。
  - 其他字段（名称、分类、描述）均可修改。
- **交互**：点击“编辑”弹出模态框，回显当前数据，保存后更新列表。

### 3.5 删除责任
- **功能描述**：从字典中移除责任项。
- **交互**：点击“删除”需弹出二次确认对话框（`confirm`），确认后执行删除。
- **数据流**：删除操作实时更新本地列表并触发 `onSave` 回调。

### 3.6 查看详情
- **功能描述**：在只读模态框中查看责任项的完整信息。
- **场景**：用于查看被截断的长描述信息，或在不进入编辑态的情况下确认内容。

## 4. 数据模型 (Data Model)

### 4.1 实体：ResponsibilityItem
| 字段名 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| `id` | string | 是 | 唯一标识符（如 `resp-1715...`） |
| `code` | string | 是 | 业务编码（如 `DEATH_BENEFIT`） |
| `name` | string | 是 | 展示名称（如 `身故保险金`） |
| `category` | string | 是 | 所属分类（来源于 `LEVEL_1_DATA`） |
| `description` | string | 否 | 详细描述 |

### 4.2 关联数据
- **LEVEL_1_DATA**：一级分类字典，用于“责任分类”下拉选项。
  - 示例值：`医疗险`, `重疾险`, `意外险`, `养老金`, `储蓄型`, `定期寿险`。

---

## 5. 界面与交互设计 (UI/UX)

### 5.1 布局结构
- **顶部**：标题“责任管理”。
- **工具栏**：
  - 左侧：搜索输入框 + “搜索”按钮。
  - 右侧：“添加责任”主操作按钮。
- **主体**：数据表格（Table）。
  - 表头背景色：浅灰 (`#fafafa`)。
  - 斑马纹或悬停高亮：提升阅读体验。
- **弹窗**：使用模态框（Modal/Overlay）承载新增、编辑和详情视图。

### 5.2 交互反馈
- **新增/编辑**：弹窗表单，底部由“取消”和“确定”按钮。
- **删除**：浏览器原生 `confirm` 弹窗或自定义 Confirm Modal。
- **状态同步**：所有增删改操作完成后，需调用组件 Props 中的 `onSave(newList)` 方法，将最新数据回传给父组件以确保持久化或全局状态更新。

## 6. 技术实现要点 (Technical Implementation)

- **组件路径**：`components/ResponsibilityManagementPage.tsx`
- **状态管理**：
  - 使用 `useState` 管理内部列表数据 `list`。
  - 使用 `useMemo` 处理搜索过滤逻辑 `filtered`。
- **Props 接口**：
  ```typescript
  interface Props {
    responsibilities?: ResponsibilityItem[]; // 初始数据
    onSave?: (list: ResponsibilityItem[]) => void; // 数据变更回调
  }
  ```
- **初始化逻辑**：
  - 若传入 `responsibilities`，则使用传入数据初始化。
  - 若未传入，则加载默认 Mock 数据 (`MOCK_RESPONSIBILITIES`) 以便演示或开发。
- **样式方案**：Tailwind CSS。

## 7. 验收标准 (Acceptance Criteria)

1.  **初始化**：页面加载时能正确显示现有的责任列表。
2.  **搜索**：输入“医疗”能筛选出所有名称或代码中包含“医疗”的条目。
3.  **新增**：
    - 点击添加能打开弹窗。
    - 填写完整信息后点击确定，列表首行出现新条目。
    - 责任分类下拉选项完整。
4.  **编辑**：
    - 点击编辑回显数据正确。
    - 责任代码置灰不可改。
    - 修改名称后保存，列表即时更新。
5.  **删除**：
    - 点击删除弹出确认框。
    - 确认后条目消失。
6.  **数据回调**：任何数据变更操作均触发 `onSave`，且参数为变更后的全量列表。
