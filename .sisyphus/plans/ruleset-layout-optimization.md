# 规则集画布布局优化计划

## 问题诊断
当前布局的问题：
1. 三个域（定责/定损/后处理）节点混在一起，没有清晰区隔
2. 层级关系不直观，难以一眼看出执行流程
3. 域、类别、规则没有明确的分组

## 目标
实现**泳道式布局**，让三个域像三条独立的流水线一样清晰区分

## 设计蓝图

### 布局架构

### 视觉设计

#### 泳道样式
- **定责域（ELIGIBILITY）**：蓝色主题 `#dbeafe` 背景 + `#3b82f6` 边框
- **定损域（ASSESSMENT）**：绿色主题 `#dcfce7` 背景 + `#22c55e` 边框  
- **后处理域（POST_PROCESS）**：紫色主题 `#f3e8ff` 背景 + `#a855f7` 边框

#### 连接边颜色
- 规则集 → 各域：使用对应域的主题色
- 域 → 类别：使用对应域的主题色（透明度50%）
- 类别 → 规则：灰色

#### 布局参数
- 泳道宽度：400px
- 泳道间距：60px
- 垂直间距：
  - 规则集到域：120px
  - 域到类别：140px
  - 类别之间：30px
  - 规则之间：15px

### 修改文件

### 关键变更
1. **移除**自动分层布局算法 `calculateLayout`
2. **改为**手动计算三泳道坐标
3. **添加**泳道背景标记节点（半透明色块）
4. **修改**边线颜色，与所在域颜色一致

### 坐标计算逻辑
```typescript
// 三个域水平排列，x坐标基于索引计算
const laneWidth = 400;
const laneGap = 60;
const domainIndex = 0/1/2; // 定责/定损/后处理
const laneX = (domainIndex - 1) * (laneWidth + laneGap);

// 垂直布局：域 → 类别 → 规则
const domainY = 120;
const categoryY = 260;
const ruleStartY = 380;
```
