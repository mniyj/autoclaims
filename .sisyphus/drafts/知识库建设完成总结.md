## 功能清单

### 主数据管理（Week 1）

#### 药品管理
- 通用名、商品名、剂型、规格管理
- 剂量范围（dose_min/max）- 医学合理性评估必需
- 疗程范围（course_min/max）- 医学合理性评估必需
- 医保类别（甲/乙/丙/自费）
- 别名管理
- 厂家、批准文号、医保编码

#### 诊疗项目管理
- 标准名称、地方名称管理
- 价格范围（price_low/high）
- 频次范围（frequency_min/max）- 医学合理性评估必需
- 疗程范围
- 适用科室
- 门诊/住院标识

#### 疾病管理
- 标准名称、ICD编码
- 严重程度（轻微/一般/严重/危重）
- 住院必要性标识
- 典型住院天数范围
- 常见检查/治疗/药品

### 别名映射（Week 2）

- 别名文本到标准实体的映射
- 实体类型支持（药品/项目/疾病/医院）
- 置信度评估和可视化
- 审核流程（待审核/已通过/已驳回）
- 标准化测试工具（实时测试）
- 批量导入支持

### 规则管理（Week 2）

5类医学合理性规则：
1. **诊断-药品匹配规则** - 判断药品与诊断是否匹配
2. **诊断-项目匹配规则** - 判断检查项目与诊断是否匹配
3. **剂量/疗程合理性规则** - 评估用药剂量合理性
4. **项目频次合理性规则** - 评估检查频次合理性
5. **住院必要性规则** - 评估住院天数合理性

每类规则支持：
- 条件表达式编辑
- 动作选择（通过/转人工/拒绝/警告）
- 优先级设置
- 原因代码

### 关系图谱（Week 3）

4类关系管理：
1. **疾病-药品关系** - 推荐/可选/不推荐/禁忌
2. **疾病-项目关系** - 常见检查和治疗
3. **手术组合关系** - 手术配套项目
4. **产品覆盖关系** - 产品责任覆盖范围

支持证据等级和置信度管理。

### 版本管理（Week 4）

- 版本历史记录
- 版本比对（Diff视图）
- 一键发布
- 一键回滚
- 变更摘要
- 版本统计

1. 启动应用：`npm run dev`

3. 左侧菜单：**理赔管理** → **知识库管理**
