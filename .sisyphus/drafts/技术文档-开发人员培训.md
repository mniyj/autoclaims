## 1. 项目概述

### 1.1 什么是保险理赔系统？

这是一个**双应用架构**的保险行业SaaS平台：

| 应用 | 中文名称 | 功能定位 | 目标用户 |
|------|----------|----------|----------|
| **Admin System** | 保险产品配置页面 | 产品/条款/理赔案件管理 | 保险公司管理员、产品经理 |
| **SmartClaim AI** | 智能理赔助手 | AI理赔顾问、聊天助手 | 保险代理人、客户 |

### 1.2 核心功能

**管理端功能：**
- 保险产品配置（医疗、意外、重疾、寿险等）
- 保险条款库管理
- 理赔案件全流程管理
- 发票OCR识别与审核
- 医保目录匹配
- 规则引擎（Ruleset）
- 公司/用户管理

**AI助手功能：**
- 智能理赔咨询
- 材料识别与审核建议
- 金额计算
- 多轮对话交互

### 1.3 双应用关系

```
┌─────────────────────────────────────────────────────────────┐
│                    保险理赔系统                              │
├───────────────────────────┬─────────────────────────────────┤
│   Admin System (8080)     │   SmartClaim AI (8081)          │
├───────────────────────────┼─────────────────────────────────┤
│  • 产品管理               │  • AI聊天助手                   │
│  • 理赔处理               │  • 材料上传                     │
│  • 发票审核               │  • 智能建议                     │
│  • 规则配置               │                                 │
├───────────────────────────┴─────────────────────────────────┤
│              共享后端服务 (Express)                          │
│  • AI引擎 (LangGraph)    • 文件存储 (OSS)                    │
│  • OCR识别               • 数据持久化 (JSON)                 │
└─────────────────────────────────────────────────────────────┘
```

### 5.5 类型安全

#### 严格类型使用

```typescript
// ✅ 明确定义类型
interface Product {
  id: string;
  name: string;
  price: number;
}

// ✅ 使用类型守卫
function isProduct(obj: unknown): obj is Product {
  return (
    typeof obj === 'object' &&
    obj !== null &&
    'id' in obj &&
    'name' in obj
  );
}

// ❌ 避免类型断言
const product = data as Product;  // 危险

// ✅ 使用类型收窄
if (isProduct(data)) {
  // 类型安全
}
```

### 5.6 错误处理

```typescript
// ✅ 使用try-catch + 具体错误
const fetchWithError = async () => {
  try {
    const result = await api.products.list();
    return result;
  } catch (error) {
    if (error instanceof Error) {
      console.error('获取产品列表失败:', error.message);
      // 显示用户友好的错误
```

### 5.7 禁止事项

| 禁止项 | 原因 | 替代方案 |

### 6.2 数据持久化

#### JSON文件存储

```
jsonlist/
├── products.json          # 产品数据

├── claim-cases.json      # 理赔案件
```

**特点：**
- 无需数据库，部署简单
- 每个资源一个JSON文件
- 通过 `readData()` / `writeData()` 操作
- 适合中小规模数据

#### 文件操作示例

```javascript
// server/utils/fileStorage.js
const readData = (resource) => {
  const filePath = path.join('jsonlist', `${resource}.json`);
  const data = fs.readFileSync(filePath, 'utf-8');
  return JSON.parse(data);
};

const writeData = (resource, data) => {
  const filePath = path.join('jsonlist', `${resource}.json`);
  fs.writeFileSync(filePath, JSON.stringify(data, null, 2));
};
```

```bash
./deploy.sh <服务器IP>

# 完整参数
./deploy.sh <服务器IP> [用户名] [SSH密钥路径] [端口]

# 示例
./deploy.sh 121.43.159.216
./deploy.sh 121.43.159.216 root ~/.ssh/id_rsa 3005
```

**部署流程：**

```
1. 本地构建 → npm run build
2. 打包 → deploy.tar.gz (dist/, server/, jsonlist/, package.json)
3. 上传 → SCP到远程服务器
4. 解压 → 在服务器上解压
5. 安装 → npm install --production
6. 重启 → PM2重启进程
7. 防火墙 → firewalld配置
```

## 10. 常见问题

### Q1: 启动开发服务器时提示端口被占用？

```bash
# 查找占用8080的进程
lsof -i :8080

# 结束进程
kill -9 <PID>

# 或修改.env.local使用其他端口
DEV_PORT=8081
```

### Q2: AI功能不工作（发票识别失败）？

```bash
# 检查环境变量
cat .env.local | grep GEMINI

# 测试API密钥
curl "https://generativelanguage.googleapis.com/v1beta/models?key=$GEMINI_API_KEY"
```

### Q3: 文件上传失败？

```bash
# 检查OSS配置
cat .env.local | grep ALIYUN

# 确认网络可以访问阿里云OSS
ping oss-cn-beijing.aliyuncs.com
```

### Q4: 如何添加新的产品类型？

1. **添加类型定义** (`types.ts`)
2. **添加表单组件** (`components/product-form/`)
3. **添加Schema** (`schemas/`)
4. **更新Mock数据** (`constants.ts`)
