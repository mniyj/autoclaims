# 现有页面功能 vs 知识库需求 - 对比分析

## 总体结论

**部分满足，部分需要扩展**

| 模块 | 满足度 | 说明 |
|------|--------|------|
| 药品管理 | ⚠️ 70% | 基本字段满足，缺少剂量/疗程、关系图谱 |
| 诊疗项目管理 | ⚠️ 70% | 基本字段满足，缺少频次规则、关系图谱 |
| 医院管理 | ⚠️ 80% | 基本字段满足，缺少别名映射、关系图谱 |
| 疾病管理 | ❌ 0% | 现有系统**完全缺失** |
| 别名映射 | ⚠️ 30% | 现有系统有aliases字段，但非独立映射表 |
| 规则管理 | ❌ 0% | 现有系统**完全缺失** |
| 关系图谱 | ❌ 0% | 现有系统**完全缺失** |

## 详细对比

| 字段 | 现有系统 | 知识库需求 | 状态 |
|------|----------|------------|------|
| id | ✅ id | ✅ drug_id | ✅ 满足 |
| 名称 | ✅ name | ✅ generic_name | ✅ 满足 |
| 商品名 | ✅ - | ✅ brand_name | ⚠️ 需添加字段 |
| 别名 | ✅ aliases | ✅ aliases | ✅ 满足 |
| 剂型 | ✅ dosageForm | ✅ dosage_form | ✅ 满足 |
| 规格 | ✅ specifications | ✅ spec | ✅ 满足 |
| 厂家 | ✅ manufacturer | ✅ manufacturer | ✅ 满足 |
| 医保类别 | ✅ type | ✅ reimbursement_flag | ✅ 满足 |
| 报销比例 | ✅ reimbursementRatio | ✅ - | ✅ 满足 |
| 限制条件 | ✅ restrictions | ✅ reimbursement_restriction | ✅ 满足 |

#### 缺少的关键功能 ❌
| 功能 | 知识库需求 | 现有系统 | 优先级 |
|------|------------|----------|--------|
| 剂量范围 | dose_min, dose_max | ❌ 无 | **P0** - 超量识别必需 |

**结论**: 基本数据管理满足，但**医学合理性评估必需**的剂量/疗程数据缺失。

### 5. 别名映射 (AliasMapping)

#### 现有系统支持 ⚠️
现有医保目录和医院管理都有 `aliases` 字段，但：
- ✅ 可以存储多个别名
- ❌ 没有**置信度**评估
- ❌ 不是**独立可维护**的映射表
- ❌ 没有**审核流程**

| 功能 | 知识库需求 | 现有系统 | 状态 |
|------|------------|----------|------|
| 别名文本 | alias_text | ✅ aliases[] | ⚠️ 基本满足 |
| 实体类型 | entity_type | ❌ 无 | ❌ 必需 |
| 实体ID | entity_id | ❌ 无 | ❌ 必需 |
| 置信度 | confidence | ❌ 无 | ❌ 必需 |
| 来源 | source | ❌ 无 | P1 |
| 审核状态 | status | ❌ 无 | P1 |

**结论**: 需要**新建独立的别名映射管理功能**。

## 建议方案

### 短期（1-2周）- MVP

**复用现有页面 + 最小化扩展**

1. **复用医保目录管理**
   - 扩展字段：dose_min, dose_max, frequency_min, frequency_max
   - 新增Tab：用于录入这些规则数据

2. **复用医院信息管理**
   - 扩展字段：aliases（如需要）

3. **新建疾病管理**（简化版）
   - 只包含：disease_id, name, icd_code, inpatient_necessity_flag

4. **新建别名映射页面**（简化版）
   - 基础CRUD功能

### 中期（3-4周）- 完整版

1. **扩展所有主数据表**
   - 补全所有缺失字段

2. **新建规则管理页面**
   - 5类规则的CRUD
   - 规则测试功能

3. **新建关系管理页面**
   - 4类关系的CRUD
   - 批量导入功能

### 长期（持续优化）

1. **数据同步机制**
   - 医保目录 ↔ 知识库药品表
   - 避免数据冗余

2. **AI辅助映射**
   - 自动别名识别
   - 智能关系推荐

---

## 最终建议

**如果急需上线**：
- 复用现有页面作为**数据录入入口**
- 扩展必要字段（剂量/频次/ICD编码）
- 先实现**疾病管理和别名映射**（最小可行）

**如果要完整功能**：
- 需要**2-3周**开发时间
- 新建疾病、规则、关系管理页面
- 数据迁移和同步机制
